## CLIP模型相关

### Q1: CLIP模型的作用是什么？

CLIP模型在当前项目中作为**本地零样本多维分类器**，主要用于：

1. **三维分类**：对图片/视频帧进行服装、动作、发型三个维度的分类
2. **预分类过滤**：作为云端VLM的前置过滤器，置信度高时直接使用本地结果，节省API调用成本
3. **视频多帧分析**：对视频进行多帧CLIP分析，检测标签变化
4. **动态学习**：支持从云端VLM结果中吸收新标签

### Q2: CLIP只能输出一个标签吗？

不是。CLIP分类器返回top-k个结果（默认top_k=5），但当前只使用置信度最高的那个标签。

**多标签模式**：启用后会返回所有高于阈值的标签，例如：`JK制服_百褶裙_白丝_站姿`

### Q3: CLIP置信度为什么这么低？

当前使用的CLIP模型（ViT-B-16，LAION数据集训练）对成人内容的分类能力有限。置信度低的原因：

1. **训练数据限制**：LAION数据集不包含成人内容
2. **内容类型不匹配**：当前视频内容超出CLIP训练分布
3. **prompt设计**：当前prompt可能不够精确

**建议**：
- 降低阈值（如0.10-0.15）
- 或寻找针对特定领域微调的CLIP模型

### Q4: 如何搜索更好的CLIP模型？

**搜索关键词**：
- "NSFW CLIP model"
- "adult content CLIP classification"
- "cosplay CLIP fine-tuned"
- "LAION aesthetic CLIP"

**推荐平台**：
1. **HuggingFace**：搜索微调模型
2. **CivitAI**：有大量anime/cosplay相关模型
3. **GitHub**：搜索CLIP fine-tune项目

**模型升级建议**：
| 模型 | 参数量 | 准确率 | 内存需求 |
|------|--------|--------|----------|
| ViT-B-16 (当前) | 86M | 基准 | 低 |
| ViT-L-14 | 304M | +5-10% | 中 |
| ViT-H-14 | 632M | +10-15% | 高 |

### Q5: 不开启CLIP预分类，标签会新增吗？

**是的，标签仍然会新增**。

标签学习逻辑：
- 只要VLM返回关键词，就会学习新标签并保存到`tag_statistics.json`
- 但如果CLIP未启用，新增的标签不会被使用
- 下次开启CLIP时，会自动使用之前学习的标签

## 视频帧选择策略

### Q6: 选择分析的帧，位于的视频时间是随机还是固定的？

**是固定的，但有三种不同的时间策略**：

| 帧类型 | 时间策略 | 公式 |
|--------|----------|------|
| **关键帧采样** | 均匀分布 | `duration * (i+1) / (n_samples+1)` |
| **均匀采样帧** | 均匀分布 | `duration * (i+1) / (n_frames+1)` |
| **人体检测帧** | 自适应步长 | `0s, step, step*2, ...`（覆盖前1/3） |

**优化后**：跳过视频开头和结尾（避免黑屏/片头片尾）

### Q7: 发送到VLM的图片是什么？

发送到VLM的是**原始帧**（关键帧/均匀采样帧），不是人体裁剪帧。

**流程**：
1. 关键帧检测 → 原始帧
2. 人体检测 → 原始帧
3. CLIP分析 → 选择最佳帧
4. 发送到VLM → 原始帧

**注意**：人体裁剪只用于embedding变化检测，不发送给VLM。

### Q8: Embedding检测如何影响VLM帧选择？

**Embedding检测结果决定下一步流程：**

```
Embedding检测结果
       │
       ▼
┌─────────────────────────────────────────┐
│ has_significant_change?                 │
│ (是否有显著变化)                         │
└─────────────────────────────────────────┘
       │
       ├───────────────────────────────────────┐
       │                                       │
       ▼                                       ▼
┌──────────────┐                       ┌──────────────┐
│ False        │                       │ True         │
│ (无显著变化)  │                       │ (有显著变化)  │
└──────────────┘                       └──────────────┘
       │                                       │
       ▼                                       ▼
┌─────────────────────┐               ┌─────────────────────┐
│ + CLIP置信度高？     │               │ 选择多帧送入VLM     │
│ (>= threshold)      │               │ (按CLIP置信度排序)   │
└─────────────────────┘               └─────────────────────┘
       │                                       │
       ├───────────┐                           │
       │           │                           │
       ▼           ▼                           ▼
   ┌──────┐   ┌──────┐                   ┌──────────┐
   │ 是   │   │ 否   │                   │ clip+cloud│
   └──────┘   └──────┘                   └──────────┘
       │           │                           │
       ▼           ▼                           ▼
┌──────────┐ ┌──────────┐               ┌──────────────┐
│clip_only │ │clip+cloud│               │ 发送到VLM    │
│跳过VLM   │ │发送到VLM │               │ 提取关键词   │
└──────────┘ └──────────┘               └──────────────┘
```

**总结表格：**

| Embedding结果 | CLIP置信度 | 下一步操作 |
|---------------|------------|------------|
| **无变化** | 高 | **跳过VLM**，直接用CLIP结果 |
| **无变化** | 低 | **发送到VLM**，用CLIP选择最佳帧 |
| **有变化** | 任意 | **发送到VLM**，用CLIP选择多帧 |

**核心作用**：Embedding检测判断视频内容是否变化，决定是否需要调用昂贵的云端VLM API。

### Q9: 纯色帧检测如何工作？

**检测方法**：计算图像灰度值的标准差

```python
def is_solid_color_frame(image_path: str, threshold: float = 15.0) -> bool:
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    std_dev = np.std(gray)
    return std_dev < threshold  # 标准差低于阈值认为是纯色
```

**阈值建议**：
- `threshold=10`：过滤纯黑/纯白
- `threshold=20`：过滤接近纯色的暗帧/亮帧
- `threshold=5`：只过滤完全纯色

## Embedding变化检测

### Q10: Embedding检测和标签比较有什么区别？

| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **标签比较** | 比较CLIP分类结果的标签集合 | 简单直观 | 依赖CLIP分类准确率 |
| **Embedding检测** | 比较人体区域的CLIP embedding向量 | 更准确，不依赖分类结果 | 需要额外的人体裁剪 |

**当前方案**：Embedding检测作为主要方式，标签比较作为备用方案。

### Q11: Embedding阈值如何设置？

| 阈值 | 敏感度 | 适用场景 |
|------|--------|----------|
| 0.60 | 非常敏感 | 检测细微变化 |
| 0.75 | 稳健（默认） | 平衡准确性和召回率 |
| 0.90 | 宽松 | 只检测显著变化 |

**建议**：使用默认值0.75，稳健且准确。

## 无意义标题检测

### Q12: 哪些标题会被标记为无意义？

以下类型的标题会被标记为`needs_vision=true`：

| 规则 | 示例 |
|------|------|
| IMG/VID/DCIM前缀 | `IMG_7940`、`VID_20240115` |
| Telegram/TG且无中文 | `Telegram@ciyuanb@-merged-...` |
| 纯hex/hash | `25bdc148`、`6330BC93-...` |
| 纯数字/日期 | `20240115`、`25-05-08` |
| 中文日期格式 | `4月23日 (7)`、`12月1日` |
| 短标题 | `视频`、`视3频`、`video (1)` |
| 视频+数字 | `视334频`、`视频 (5)` |
| **#tag标题** | `#Cosplay_女仆装`（**不会**被标记） |

### Q13: 短标题检测规则是什么？

```python
# 核心文字 = 去除括号、数字、分隔符后的文字
core_text = re.sub(r'[\(\)\[\]【】（）\d\-_.\s]+', '', name_without_ext)

# 规则1：核心文字为空或只有1-2个字符
if core_len <= 2:
    is_too_short = True

# 规则2：纯中文且只有1-2个字（如"視頻"）
if cn_count > 0 and en_count == 0 and cn_count <= 2:
    is_too_short = True

# 规则3：纯英文/数字混合且长度很短
if cn_count == 0 and core_len <= 4:
    is_too_short = True
```

**例外**：以`#`开头的标题不会被标记为无意义。

### Q14: [未分类]的文件会被送入VLM吗？

**是的，[未分类]的文件会被送入VLM**。

逻辑：
1. 阶段1：标题无有效关键词 → 标记为`[未分类]`，设置`needs_vision=true`
2. 阶段1c：只处理`needs_vision=true`的记录 → 送入VLM

## 临时目录说明

### Q17: `logs/_vision_tmp`是什么？

**用途**：Stage1c视觉识别的临时文件目录
- 存储视频帧提取的临时图片
- 存储CLIP分析的中间帧
- 存储关键帧检测的采样帧
- 处理完成后自动清理

### Q18: `logs/_frame_selector_tmp`是什么？

**用途**：人体检测模块的临时文件目录
- 存储人体检测过程中提取的帧
- 存储找到的人体帧
- 用于后续embedding检测和VLM调用

**注意**：这两个目录都在`.gitignore`中，不会被提交到Git。

## 常见问题

### Q19: 为什么图片模式没有被处理？

**可能原因**：
1. 使用了`--single`参数只处理特定文件
2. 图片的`needs_vision`不是`true`
3. 图片压缩失败
4. VLM API调用失败

**解决方法**：
```bash
# 处理所有未处理的文件
uv run python stage1c_vision_refine.py --retry-errors

# 强制处理所有文件
uv run python stage1c_vision_refine.py --all
```

### Q20: 为什么CLIP置信度为0.0？

**原因**：CLIP分类返回的置信度低于阈值（默认0.25），被过滤掉了。

**解决方法**：
1. 降低阈值：`--clip-threshold 0.10`
2. 检查CLIP模型是否正确加载
3. 检查`models/clip/`目录是否有模型文件

### Q21: 为什么视频处理中断了？

**可能原因**：
1. 手动停止了进程
2. API调用超时
3. 磁盘空间不足
4. 文件被占用

**解决方法**：
```bash
# 重试失败的记录
uv run python stage1c_vision_refine.py --retry-errors
```

### Q22: 如何提高处理速度？

**优化建议**：
1. 减少CLIP分析帧数：`--clip-frames 3`
2. 减少VLM帧数：`--vlm-frames 2`
3. 减少关键帧数：`--max-keyframes 5`
4. 使用GPU加速（如果可用）
5. 排除不需要的目录：`--exclude-dir temp`

### Q23: 如何撤销重命名？

**方法1**：从CSV的`original_title`列手动还原

**方法2**：执行前备份目录
```powershell
robocopy "F:\Videos" "F:\Videos_Backup" /E /NFL /NDL /NJH /NJS
```

### Q24: CSV在Excel中打开乱码

**原因**：编码不是UTF-8 with BOM。

**解决方法**：
1. 用VS Code打开CSV
2. 右下角点击编码 → "通过编码保存"
3. 选择"UTF-8 with BOM"
4. 重新用Excel打开
